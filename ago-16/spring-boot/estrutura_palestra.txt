
Palestra SpringBoot

O que é 
	- Visão geral
		- Quando surgiu 
			- Começou no final de 2013
			- 1.0 começo de 2014
		- O que é
			- Meta framework (ou framework de frameworks), que ajuda à iniciar as aplicações sem se preocupar com muitas configurações (principalmente xml)
			- Provê uma maneira muito mais rápida de começar a construir aplicações runnable java baseadas em spring. 
			- Provê várias features não funcionais e production ready que são muito usadas, como os embbeded servers, metrics, health, security.
			- Automaticamente configura as outras dependencias do spring
			- Vários starter dependencies pras bibliotecas mais famosas
			- Não tem necessidade de xml
			- externalização de configurações (app.properties)
				Tem a opção de usar yml (app.yml)
Que problemas resolve / Porque usar
	- Mostrar facilidade de inicio
	- https://start.spring.io/
	- a necessidade de um melhor bootstrap
	- O que difere do spring framework convencional
	- runnable jar (fatjar) mas também suporta o tradicional deployar em application container
Como funciona
	- auto configuration
		- disable specific auto configuration * lembrar de mencionar
		- troubleshoot auto configuration
	- classpath mapping de properties/xml
	- maven e gradle plugin
	- Últimas versões é java 7+, porém é possível utilizar com java 6 com algumas configurações
Algumas features
	- dependencies starter (mais famosas)
		- embbeded servlet container
		- production ready features - endpoints de erro, healthcheck, metricas de aplicação e jvm, jmx via http
		- thymeleaf
		- jax-rs / jersey
		- jackson mapping
		- logger
		- security
		- filas
	- profiling
	- comandos básicos (como startar)
	- falar rapido sobre spring boot cli
	- buildar e deployar
